# 科研技能路由地图

本地图用于复杂科研任务的点将。它只提供调度建议，不要求用户安装固定 skill；主代理应先读取当前可见 skills metadata，再选择最匹配的主将和副将。

## 路由原则

1. 先定交付物，再定主将。
2. 一个阶段只设一个主将；其他能力作为副将或监军检查。
3. 已有 skill 覆盖大部分需求时，优先组合现有能力，不默认联网征召。
4. 本地没有合适主将时，让斥候输出征召令，不直接安装高风险候选。
5. 科研任务必须保留证据链、引用来源、数据边界和不确定性。

## 未来 skill 纳入规则

新增 skill 不需要修改本地图才能被使用。主代理应读取它的 name、description、触发描述和可见路径，并按以下问题判断：

- 它能否承担本阶段交付物的核心规则。
- 它是否比现有主将更贴合用户任务。
- 它是否涉及隐私外发、凭据、账号写入、批量整理或删除。
- 它是否需要额外软件、许可证、登录或网络访问。
- 它的输出能否被监军用测试、渲染、引用、日志或人工核查验证。

## 防侵权边界

本地图只描述能力类别和调度方式。不要把第三方 skill 的说明书、脚本、表格或示例复制进来；需要提到外部项目时，只写项目名、能力类别和安全边界。
