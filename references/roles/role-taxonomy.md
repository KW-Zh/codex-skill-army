# 军制角色表

本文件只在任务复杂、跨领域、需要多 skill 或子代理时加载。简单问答不要加载完整军制。

## 角色与边界

| 角色 | 军中职责 | 实际含义 | 不应做什么 |
| --- | --- | --- | --- |
| 总帅 | 定目标、点将、分兵、合围、验收 | `codex-skill-army` 本体 | 不替代领域 skill 硬做专业任务 |
| 军师 | 计划、拆解、取舍 | 澄清目标、选择路线、定义验收 | 不把所有任务都流程化 |
| 斥候 | 找兵、审兵 | 查本地 skills，搜索高分候选，输出征召令 | 不绕过用户确认直接安装高风险工具 |
| 主将 | 攻坚交付 | 最匹配的领域 skill | 不同时指定多个主将导致责任不清 |
| 副将 | 补位支援 | 检索、图表、格式、测试、部署等辅助 skill | 不堆砌无关技能 |
| 监军 | 查质量和风险 | 测试、引用核验、隐私、安全、发布检查 | 不用主观评价代替证据 |
| 军需官 | 管能力地图 | 盘点技能库、查重复、查缺口、查悬空引用 | 不批量改写所有 skill |
| 史官 | 记长期经验 | 保存研究偏好、项目决策、复用经验 | 不默认记录隐私、凭据、原始全文 |
| 工部 | 管基础设施 | CodeGraph、MCP、CLI、安装器、脚本、CI | 不为简单任务引入重工具 |

## 点将顺序

1. 先定交付物，再定角色。
2. 一个阶段只选一个主将；副将最多 4 个。
3. 主将缺失时，让斥候出征召令。
4. 长期项目才考虑史官；短任务不写记忆。
5. 所有最终判断由总帅合围，监军按证据验收。

## 科研常见点将

| 场景 | 主将 | 常见副将 | 监军重点 |
| --- | --- | --- | --- |
| 文献综述 | 文献检索/阅读 skill | 引用、写作、表格 | 来源可信、引用准确 |
| Word 论文 | Word/Office skill | 公式、参考文献、PDF | 排版、编号、交叉引用 |
| Nature 风格写作 | Nature/Scientific writing skill | citation、figure、response | 论证链、期刊风格 |
| 科研代码库 | CodeGraph 或代码分析 skill | 测试、安全、文档 | 架构证据、测试结果 |
| Abaqus/FEA | Abaqus 领域 skill | 材料、网格、载荷、ODB | 边界条件、单位、结果提取 |
| 缺少领域能力 | 斥候 | 军需官 | 来源、安全、维护状态 |
