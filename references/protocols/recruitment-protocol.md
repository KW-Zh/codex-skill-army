# 缺兵征召协议

当本地 skill 库没有合适主将或副将时使用。目标是补足能力，不是凑数量。

## 本地优先

1. 先查看当前会话可见 skills metadata。
2. 必要时只读扫描本地 skills 目录。
3. 已有 skill 能覆盖 70% 以上需求时，优先组合已有能力，不征召。

## 外部搜索顺序

1. 官方或可信组织维护的 skills。
2. 高星、近期维护、路径含 `SKILL.md` 的 GitHub 仓库。
3. 与任务领域高度匹配的小而专技能。
4. 找不到可信候选时，设计最小专用 skill。

## 评分

满分 100：

- 相关性 40：是否直接覆盖缺口。
- 来源可信度 25：官方、高星、维护活跃、issue 状态。
- Skill 完整性 20：`SKILL.md`、references/scripts、清晰触发描述。
- 安全性 15：无默认隐私外发、凭据写入、批量整理或删除行为。

75 分以上推荐安装；60 到 74 分列为备选；60 分以下不列入征召令。

## 征召令模板

```text
缺口：
本地结果：
候选排序：
风险：
建议：
安装或创建命令：
校验方式：
是否需要用户确认：
```

## 安全排除

默认排除：

- 默认读取 Codex 历史、聊天记录、邮件、浏览器数据。
- 默认批量整理、移动或删除用户文件。
- 要求上传凭据或外发私密数据。
- 需要付费账号才能完成核心功能，但未清楚说明。
- 无许可证、无维护迹象、无 `SKILL.md` 的伪 skill。
