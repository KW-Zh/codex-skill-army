# 科研代码库工作流

## 触发

用户要求理解大型代码库、改科研软件、复现实验、检查数据处理 pipeline、定位模型训练或分析脚本。

## 点将

- 工部：CodeGraph 优先用于结构、符号、依赖、影响面。
- 主将：代码语言或框架相关 skill。
- 副将：测试、数据、可视化、安全、文档。
- 监军：运行最小测试、复现实验或静态检查。

## 流程

1. 先确认项目入口、运行方式、数据要求。
2. 大型代码库优先用 CodeGraph 或已有索引。
3. 环境锁定：记录 Python/R/Matlab/Julia 版本、依赖文件、系统库、GPU/CUDA 条件。
4. 识别数据边界：区分公开数据、私有数据、临时路径、脱敏样例和不可提交材料。
5. 不在不了解数据含义、单位、筛选条件和统计假设时修改科研结论。
6. 改动后保留复现命令、随机种子、配置文件、日志位置和关键输出。
7. Notebook 任务要清理大输出、绝对路径、隐私数据和不可复现的交互状态。
8. 结果图和表格要能追溯到脚本、输入数据和参数。
9. 未能运行测试或复现实验时，说明阻塞条件和最小替代验证。

## 验收清单

- 是否有 `pyproject.toml`、`requirements.txt`、`environment.yml`、`renv.lock` 或等价环境说明。
- 是否记录随机种子、数据版本、训练/分析配置和输出目录。
- 是否存在硬编码个人路径、私有数据路径、API key 或凭据；必要时做数据路径脱敏。
- 是否运行单元测试、最小复现实验、静态检查或关键 notebook。
- 是否说明结果变动对论文结论、图表和补充材料的影响。

## 风险

- 数据泄露。
- 随机性导致结果不可复现。
- 单位、坐标系、样本筛选条件被误改。
- 结果图看似正常但统计逻辑错误。
- 大文件、模型权重、原始数据或 notebook 输出被误提交。
