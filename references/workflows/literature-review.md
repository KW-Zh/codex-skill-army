# 科研文献综述工作流

## 触发

用户要求读论文、查文献、做综述、写研究现状、比较方法、生成中文汇报或准备开题材料。

## 点将

- 军师：先定义研究问题、范围、时间跨度和输出格式。
- 主将：论文检索/阅读/综述类 skill。
- 副将：引用核验、科学写作、Markdown/Mermaid、Word/PDF。
- 监军：检查来源、引用格式、事实不确定性。
- 史官：保存研究方向、筛选标准、用户偏好。

## 最小流程

1. 明确主题、关键词、纳入/排除标准。
2. 选择数据库：Web of Science、Scopus、PubMed、IEEE Xplore、arXiv、Google Scholar、CNKI 等按领域取舍。
3. 写检索式：列出英文关键词、中文关键词、同义词、布尔逻辑和时间范围。
4. 去重和筛选：记录去重规则、纳入标准、排除标准、低质量文献剔除理由。
5. DOI/引用核验：保留 DOI、URL、年份、期刊/会议、检索日期。
6. 证据分组：按方法、数据、结论、局限、适用范围分组。
7. 输出结构化综述，而不是论文摘要堆叠。
8. 标注未核验或证据不足的判断。

## 系统综述字段

关键文献表至少包含：

| 字段 | 说明 |
| --- | --- |
| 题名 | 原始题名 |
| 作者年份 | 第一作者和年份 |
| DOI/URL | 可追溯来源 |
| 数据/样本 | 实验、仿真或公开数据来源 |
| 方法 | 模型、算法、试验或理论方法 |
| 主要结论 | 与研究问题相关的结论 |
| 局限 | 样本、假设、适用范围 |
| 纳入理由 | 为什么保留 |

PRISMA 式流程可用简表表示：检索数量、去重后数量、标题摘要筛选数量、全文筛选数量、最终纳入数量。

## 交付物

- 中文综述。
- 关键文献表。
- 检索式和数据库清单。
- 纳入/排除记录。
- 研究空白。
- 后续实验或写作建议。
